🖥️ ما هو Back-End؟

Back-End

الجزء الخفي من التطبيق الذي يعمل على الخادم ويدير البيانات والمنطق

تخيل مطعماً: Front-End هو صالة الطعام التي يراها الزبائن، أما Back-End فهو المطبخ حيث يتم تحضير الطعام وإدارة المخزون - لا يراه الزبائن لكنه أساسي لعمل المطعم!

مثال من الحياة: عندما تسجل دخولك في فيسبوك، Front-End يعرض لك الصفحة، لكن Back-End يتحقق من كلمة المرور ويجلب منشوراتك!

Front-End vs Back-End:

🎨 Front-End

  • ما يراه المستخدم
  • HTML, CSS, JavaScript
  • التصميم والتفاعل
  • يعمل في المتصفح
  • واجهة المستخدم

⚙️ Back-End

  • ما لا يراه المستخدم
  • Python, PHP, Node.js
  • المنطق وقواعد البيانات
  • يعمل على الخادم
  • معالجة البيانات

🖥️ ما هي الخوادم (Servers)؟

الخادم (Server)

حاسوب قوي يعمل 24/7 لخدمة طلبات المستخدمين وتشغيل التطبيقات

🌐
Web Server

خادم الويب

يقدم ملفات HTML, CSS, JavaScript للمتصفحات

أمثلة: Apache, Nginx

⚙️

🔌 ما هي APIs؟

API

Application Programming Interface
واجهة تسمح للتطبيقات بالتواصل مع بعضها البعض

🍕 مثال: طلب البيتزا

تخيل أنك تطلب بيتزا بالهاتف. أنت لا تحتاج لمعرفة كيف تُصنع البيتزا، فقط تطلب ما تريد والمطعم يحضرها لك. API يعمل بنفس الطريقة!

  • أنت: التطبيق الذي يطلب البيانات
  • القائمة: API Documentation
  • الطلب: HTTP Request
  • البيتزا: البيانات المطلوبة

// مثال على استخدام API بـ JavaScript async function getWeatherData(city) { try { // إرسال طلب GET للـ API const response = await fetch(`https://api.weather.com/v1/current?city=${city}`); // تحويل الاستجابة إلى JSON const data = await response.json(); // استخدام البيانات console.log(`الطقس في ${city}: ${data.temperature}°C`); return data; } catch (error) { console.error('خطأ في جلب بيانات الطقس:', error); } }

📝 مثال بسيط: خادم Node.js

إنشاء خادم بسيط بـ Node.js:

// تثبيت Node.js أولاً، ثم إنشاء ملف server.js const http = require('http'); const url = require('url'); // إنشاء الخادم const server = http.createServer((req, res) => { // إعداد headers res.setHeader('Content-Type', 'application/json; charset=utf-8'); res.setHeader('Access-Control-Allow-Origin', '*'); // تحليل الـ URL const parsedUrl = url.parse(req.url, true); const path = parsedUrl.pathname; const method = req.method; // API endpoints if (path === '/api/hello' && method === 'GET') { res.statusCode = 200; res.end(JSON.stringify({ message: 'مرحباً من الخادم!', timestamp: new Date().toISOString() })); } else if (path === '/api/users' && method === 'GET') { res.statusCode = 200; res.end(JSON.stringify({ users: [ { id: 1, name: 'أحمد', email: 'ahmed@email.com' }, { id: 2, name: 'فاطمة', email: 'fatima@email.com' } ] })); } else { res.statusCode = 404; res.end(JSON.stringify({ error: 'الصفحة غير موجودة' })); } }); // تشغيل الخادم const PORT = 3000; server.listen(PORT, () => { console.log(`الخادم يعمل على http://localhost:${PORT}`); });

تشغيل الخادم:

# في Terminal node server.js # ثم اذهب إلى: # http://localhost:3000/api/hello # http://localhost:3000/api/users

  • لا يوجد فرق
  • Front-End يراه المستخدم، Back-End لا يراه
  • Front-End أصعب من Back-End
  • Back-End يستخدم HTML فقط
