🔄 ما هو Git؟

Git

نظام إدارة الإصدارات الموزع - يتتبع التغييرات في ملفاتك عبر الزمن

تخيل أنك تكتب مقالاً مهماً، وتريد حفظ نسخة من كل تعديل تقوم به. Git يفعل هذا تلقائياً لكودك! يحفظ كل تغيير، ويمكنك الرجوع لأي إصدار سابق.

مثال من الحياة: مثل آلة الزمن لكودك! يمكنك السفر للماضي ورؤية كيف كان كودك قبل أسبوع أو شهر.

لماذا نحتاج Git؟

  • تتبع التغييرات: معرفة ماذا تغير ومتى وبواسطة من
  • النسخ الاحتياطية: لن تفقد كودك أبداً
  • التعاون: عدة مطورين يعملون على نفس المشروع
  • التجريب الآمن: جرب أفكار جديدة دون خوف
  • إدارة الإصدارات: v1.0, v2.0, v3.0...

⚙️ تثبيت Git

خطوات التثبيت:

  1. اذهب إلى git-scm.com
  2. حمل Git لنظام التشغيل الخاص بك
  3. شغل ملف التثبيت واتبع التعليمات
  4. افتح Terminal أو Command Prompt

التحقق من التثبيت:

في Terminal:

git --version git version 2.40.0

الإعداد الأولي:

تعيين اسمك وبريدك الإلكتروني:

git config --global user.name "اسمك" git config --global user.email "email@example.com" # التحقق من الإعدادات git config --list

دمج التغييرات من فرع إلى آخر

إنشاء مستودع جديد:

إنشاء مجلد جديد وتهيئة Git:

mkdir my-project cd my-project git init # إنشاء ملف README echo "# مشروعي الأول" > README.md
my-project/ ├── 📁 .git/ # مجلد Git (مخفي) └── 📄 README.md # ملف المشروع

⌨️ أوامر Git الأساسية

git status

الوظيفة: عرض حالة الملفات

git status On branch main Untracked files: README.md
git add

الوظيفة: إضافة ملفات للتتبع

git add README.md # أو إضافة جميع الملفات git add .
git commit

الوظيفة: حفظ التغييرات

git commit -m "إضافة ملف README"
git log

الوظيفة: عرض تاريخ التغييرات

git log --oneline a1b2c3d إضافة ملف README

سير العمل الأساسي:

مثال كامل:

# 1. إنشاء ملف جديد echo "console.log('مرحباً بالعالم!');" > app.js # 2. فحص الحالة git status # 3. إضافة الملف للتتبع git add app.js # 4. حفظ التغييرات git commit -m "إضافة ملف JavaScript الأساسي" # 5. عرض التاريخ git log --oneline

🌿 الفروع (Branches)

الفروع تتيح لك العمل على ميزات مختلفة دون التأثير على الكود الأساسي:

🌳 مثال على الفروع

main: ─ ─ ─
feature: ─

الأخضر: commits عادية | البنفسجي: فرع جديد | البرتقالي: دمج

أوامر الفروع:

إنشاء والتنقل بين الفروع:

# عرض الفروع الحالية git branch # إنشاء فرع جديد git branch feature-login # الانتقال للفرع الجديد git checkout feature-login # إنشاء والانتقال في أمر واحد git checkout -b feature-signup # العودة للفرع الرئيسي git checkout main # حذف فرع git branch -d feature-login

دمج الفروع (Merge):

دمج فرع في الفرع الرئيسي:

# الانتقال للفرع الرئيسي git checkout main # دمج الفرع git merge feature-signup # حذف الفرع بعد الدمج git branch -d feature-signup

🐙 ما هو GitHub؟

GitHub

منصة استضافة للمشاريع البرمجية باستخدام Git - مثل فيسبوك للمطورين!

🐙

GitHub = Git + Hub

مكان لحفظ ومشاركة والتعاون في المشاريع البرمجية

مميزات GitHub:

  • استضافة مجانية: لمشاريعك العامة والخاصة
  • التعاون: عمل جماعي على نفس المشروع
  • المتابعة: تتبع المشاكل والطلبات
  • النشر: GitHub Pages لنشر المواقع
  • المجتمع: ملايين المطورين والمشاريع

إنشاء حساب GitHub:

  1. اذهب إلى github.com
  2. اضغط "Sign up" وأنشئ حساب جديد
  3. تحقق من بريدك الإلكتروني
  4. أكمل إعداد الملف الشخصي

🔗 ربط Git بـ GitHub

إنشاء مستودع على GitHub:

خطوات إنشاء Repository:

  1. اضغط "New repository" في GitHub
  2. اختر اسماً للمستودع
  3. اختر عام (Public) أو خاص (Private)
  4. اضغط "Create repository"

ربط المشروع المحلي بـ GitHub:

الأوامر:

# إضافة رابط GitHub git remote add origin https://github.com/username/my-project.git # رفع الكود لأول مرة git push -u origin main # للرفع في المرات القادمة git push

تحميل مشروع من GitHub:

استنساخ مستودع:

# تحميل مشروع git clone https://github.com/username/project-name.git # الدخول للمجلد cd project-name # عرض الملفات ls -la

Pull Request:

ما هو Pull Request؟

طلب لمراجعة ودمج تغييراتك في المشروع الأساسي. مثل طلب إذن لإضافة شيء جديد!

خطوات إنشاء Pull Request:

  1. ادفع تغييراتك لفرعك على GitHub
  2. اذهب لصفحة المستودع على GitHub
  3. اضغط "Compare & pull request"
  4. اكتب وصفاً للتغييرات
  5. اضغط "Create pull request"

🚀 مشروع عملي: نشر موقعك على GitHub

سننشر موقع Portfolio الذي بنيناه سابقاً:

1. إعداد المشروع:

# إنشاء مجلد المشروع mkdir my-portfolio cd my-portfolio # تهيئة Git git init # إنشاء ملفات الموقع touch index.html style.css script.js

2. إضافة المحتوى وحفظه:

# إضافة جميع الملفات git add . # أول commit git commit -m "إضافة ملفات الموقع الأساسية" # إضافة المزيد من المحتوى git add . git commit -m "إضافة تصميم CSS" git add . git commit -m "إضافة تفاعل JavaScript"

3. رفع المشروع لـ GitHub:

# ربط بـ GitHub git remote add origin https://github.com/username/my-portfolio.git # رفع الكود git push -u origin main

4. تفعيل GitHub Pages:

  1. اذهب لإعدادات المستودع في GitHub
  2. انزل لقسم "Pages"
  3. اختر "Deploy from a branch"
  4. اختر "main" branch
  5. احفظ الإعدادات
مبروك! موقعك الآن متاح على: https://username.github.io/my-portfolio
