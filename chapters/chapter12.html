🐛 ما هو Debugging؟

Debugging

عملية اكتشاف وإصلاح الأخطاء (Bugs) في الكود البرمجي

Debugging هو فن وعلم في نفس الوقت! كل مطور يقضي جزءاً كبيراً من وقته في البحث عن الأخطاء وإصلاحها. لا تقلق، هذا طبيعي تماماً - حتى أفضل المطورين في العالم يواجهون أخطاء يومياً!

حقيقة مثيرة: كلمة "Bug" جاءت من حشرة حقيقية علقت في حاسوب عام 1947 وتسببت في خطأ! 🐛

لماذا تحدث الأخطاء؟

  • أخطاء كتابة: خطأ في بناء الجملة
  • أخطاء منطقية: الكود يعمل لكن النتيجة خاطئة
  • أخطاء وقت التشغيل: تحدث أثناء تشغيل البرنامج
  • عدم فهم المتطلبات: بناء شيء غير مطلوب
  • تغيير في البيئة: اختلاف في المتصفح أو النظام

مثال على خطأ شائع:

❌ الكود الخاطئ: function calculateTotal(price, tax) { return price + tax * price; } let total = calculateTotal(100, 0.15); console.log(total); // النتيجة: 115 ✅ // لكن ماذا لو... let total2 = calculateTotal(100, "0.15"); console.log(total2); // النتيجة: "1000.15" ❌ خطأ!
✅ الكود المُصحح: function calculateTotal(price, tax) { // التحقق من نوع البيانات if (typeof price !== 'number' || typeof tax !== 'number') { throw new Error('Price and tax must be numbers'); } return price + (tax * price); } let total = calculateTotal(100, 0.15); console.log(total); // النتيجة: 115 ✅

🔍 أنواع الأخطاء

1. Syntax Errors

أخطاء بناء الجملة

أخطاء في كتابة الكود نفسه

// خطأ: قوس مفقود
function sayHello( {
    console.log("Hello");
}

// خطأ: فاصلة منقوطة مفقودة
let name = "أحمد"
let age = 25
2. Runtime Errors

أخطاء وقت التشغيل

تحدث أثناء تشغيل البرنامج

// خطأ: متغير غير معرف
console.log(undefinedVariable);

// خطأ: استدعاء دالة غير موجودة
nonExistentFunction();

// خطأ: الوصول لخاصية في null
let obj = null;
console.log(obj.name);
3. Logic Errors

أخطاء منطقية

الكود يعمل لكن النتيجة خاطئة

// خطأ منطقي: استخدام = بدلاً من ==
let age = 18;
if (age = 21) { // خطأ! يجب أن يكون ==
    console.log("يمكنك القيادة");
}

// خطأ منطقي: حلقة لا نهائية
for (let i = 0; i < 10; i--) { // خطأ!
    console.log(i);
}
4. Type Errors

أخطاء نوع البيانات

استخدام نوع بيانات خاطئ

// خطأ: محاولة استدعاء رقم كدالة
let number = 42;
number(); // TypeError

// خطأ: محاولة الوصول لخاصية في undefined
let user;
console.log(user.name); // TypeError

🛠️ أدوات التصحيح

🖥️
Browser DevTools

أدوات المطور في المتصفح - الأداة الأهم للمطور

  • Console للرسائل والأخطاء
  • Sources للكود والـ breakpoints

  • Elements لـ HTML و CSS

  • console.log() للرسائل العادية
  • console.error() للأخطاء
  • console.warn() للتحذيرات
  • console.table() للجداول
🔴
Breakpoints

نقاط توقف لفحص الكود خطوة بخطوة

  • إيقاف التنفيذ في نقطة معينة
  • فحص قيم المتغيرات
  • تتبع مسار التنفيذ
  • التنفيذ خطوة بخطوة
🔍
Debugger Statement

كلمة debugger في الكود لإيقاف التنفيذ

  • إضافة debugger; في الكود
  • يوقف التنفيذ تلقائياً
  • مفيد للتصحيح السريع
  • لا تنس حذفه من الكود النهائي

🖥️ استخدام Console للتصحيح

أنواع رسائل Console:

رسالة عادية
> console.warn("تحذير!");

> console.error("خطأ!");

> console.info("معلومة");
ℹ️ معلومة

تقنيات Console متقدمة:

أمثلة عملية:

تقنيات Console 
// 1. طباعة متغيرات متعددة
let name = "أحمد";
let age = 25;
console.log("الاسم:", name, "العمر:", age);

// 2. استخدام Template Literals
console.log(`المستخدم: ${name}, العمر: ${age}`);

// 3. طباعة كائن بشكل جميل
let user = {name: "أحمد", age: 25, city: "الرياض"};
console.table(user);

// 4. تجميع الرسائل
console.group("معلومات المستخدم");
console.log("الاسم:", name);
console.log("العمر:", age);
console.groupEnd();

// 5. قياس الوقت
console.time("عملية حسابية");
// ... كود معقد
console.timeEnd("عملية حسابية");

// 6. عد مرات الاستدعاء
console.count("تم استدعاء الدالة");
console.count("تم استدعاء الدالة");

// 7. تتبع مسار الاستدعاء
console.trace("من أين جاء هذا الاستدعاء؟");

🎮 مثال تفاعلي: تصحيح الأخطاء

🐛 اكتشف وأصلح الخطأ!

الكود التالي يحتوي على خطأ. استخدم الأزرار لتصحيحه:

function calculateAverage(numbers) {
    let sum = 0;
    for (let i = 0; i <= numbers.length; i++) {
        sum += numbers[i];
    }
    return sum / numbers.length;
}

let scores = [85, 90, 78, 92, 88];
let average = calculateAverage(scores);
console.log("المتوسط:", average);

🔴 استخدام Breakpoints

كيفية استخدام Breakpoints:

  1. افتح Developer Tools (F12)
  2. اذهب لتبويب "Sources"
  3. اختر الملف الذي تريد تصحيحه
  4. اضغط على رقم السطر لإضافة breakpoint
  5. شغل الكود وسيتوقف عند الـ breakpoint
  6. استخدم أزرار التحكم للتنقل

أزرار التحكم في Debugger:

▶️ Continue

متابعة التنفيذ حتى الـ breakpoint التالي

⏭️ Step Over

تنفيذ السطر التالي دون الدخول في الدوال

⬇️ Step Into

الدخول داخل الدالة المستدعاة

⬆️ Step Out

الخروج من الدالة الحالية

مثال على استخدام debugger:

function complexCalculation(x, y) {
    debugger; // سيتوقف هنا
    let step1 = x * 2;
    debugger; // وهنا أيضاً
    let step2 = y + 10;
    debugger; // وهنا
    return step1 + step2;
}

let result = complexCalculation(5, 3);
console.log(result);

⭐ أفضل الممارسات

🎯 نصائح للتصحيح الفعال:

  • ابدأ بالبساطة: استخدم console.log أولاً
  • اقرأ الأخطاء بعناية: لا تتجاهل رسائل الخطأ
  • اختبر أجزاء صغيرة: لا تختبر كل شيء مرة واحدة
  • استخدم أسماء واضحة: للمتغيرات والدوال
  • اكتب تعليقات: لتذكر ما يفعله الكود
  • احتفظ بنسخ احتياطية: قبل التغييرات الكبيرة

🚫 أخطاء شائعة يجب تجنبها:

  • تجاهل رسائل الخطأ: اقرأها دائماً
  • تغيير أشياء كثيرة معاً: غير شيئاً واحداً في كل مرة
  • عدم الاختبار: اختبر بعد كل تغيير
  • الاستعجال: خذ وقتك في الفهم
  • عدم طلب المساعدة: لا تتردد في السؤال

🔍 مصادر المساعدة:

  • Google: ابحث عن رسالة الخطأ
  • Stack Overflow: أكبر مجتمع للمطورين
  • MDN Web Docs: وثائق JavaScript الرسمية
  • GitHub Issues: مشاكل المكتبات والأدوات
  • المجتمعات العربية: منتديات ومجموعات المطورين
